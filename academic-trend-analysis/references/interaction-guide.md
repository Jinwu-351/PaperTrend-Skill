# 交互指南

> 本文件定义 academic-trend-analysis skill 中哪些节点需要暂停交互、
> 哪些节点自动执行。Claude Code **必须遵循此指南**。

## 需要交互的节点（5 个）

| # | Stage | 条件 | 动作 | 恢复条件 |
|---|-------|------|------|---------|
| 1 | 0 | BM25/Milvus + 标准/快速 | 展示四合一组合选择，等待用户选 1-4 | 用户回复模式序号 |
| 2 | 0→1 | 用户选择 Milvus | 确认嵌入模型路径 + 数据库保存路径 | 用户确认或提供路径 |
| 3 | 1 内部 | WebSearch 完成 | 提 2-3 个澄清问题，等待回复 | 用户回复 |
| 4 | 1→2 | demand.json 生成 | 展示内容，等待"继续"确认 | 用户回复"继续"/"confirm" |
| 5 | 8 后 | 用户要求修订 | 收集修订请求，检查 revision_count | 用户给出具体修改意见 |

### 节点 1：模式确认（⛔ 必须 STOP）

⛔ 在此停止，等待用户回复。

展示四合一组合选项：
```
1. BM25 + 标准模式（5 层对话）     2. BM25 + 快速模式（1-2 轮澄清）
3. Milvus + 标准模式              4. Milvus + 快速模式
```
- 如果只有一种检索模式可用 → 自动选择该检索模式，仅展示对话模式（标准/快速）二选一
- 用户选择后进入 Stage 1，模式写入 `demand.json` 的 `mode` 和 `dialogue_mode` 字段

### 节点 2：Milvus 路径确认（Milvus 模式专属）

⛔ 在此停止，等待用户确认或提供路径。

**嵌入模型路径**：
- 优先检查环境变量 `MILVUS_EMBEDDING_MODEL`
- 若有 → "已检测到嵌入模型：{path}，是否使用？[Y/n]"
- 若无 → 询问用户提供（如 `BAAI/bge-base-zh-v1.5` 或本地路径）

**数据库保存路径**：
- 优先检查环境变量 `MILVUS_DATA_PATH`
- 若有 → "已检测到数据路径：{path}，是否使用？[Y/n]"
- 若无 → 询问用户提供（如 `./milvus_data`）

两个路径将写入 `demand.json` 的 `embedding_model_path` 和 `milvus_data_path` 字段。
BM25 模式下跳过此节点。

### 节点 3：需求澄清（必须 STOP）

⛔ 在此停止，等待用户回复。

- 如果用户意图已经很明确（如具体技术名词），可减少到 **1 个问题**
- 如果对话模式为 `socratic`，按 `agents/demand_analyzer.md` 的 5 层框架展开对话
- 如果对话模式为 `quick`，仅 1-2 轮 WebSearch 澄清
- 如果 **30 分钟**内未收到回复，可提醒一次后使用已有信息继续

### 节点 4：需求确认（必须 STOP）

⛔ 向用户展示 demand.json 完整内容，等待确认后再进入 Stage 2。

### 节点 5：报告修订（用户触发）

Stage 8 完成后，如果用户要求修改报告：
- ⛔ 收集具体修改意见
- 检查 `demand.json` 中的 `revision_count`（上限 2 轮）
- 如未超限，重新执行 Stage 6-8，`revision_count` 加 1
- 如已超限，告知用户并建议将剩余问题列入 "Acknowledged Limitations"

## 不需要交互的步骤（自动执行）

Stage 2-8 均为**纯自动化步骤**，完成后向用户展示结果即可：

| Stage | 完成后展示 |
|-------|-----------|
| 2 论文搜索 | 来源分布 + 去重前后数量 |
| 3 摘要存储 | 入库论文数 |
| 4 核心论文筛选 | 核心论文列表（标题 + score） |
| 5 补充检索 | 补充论文数 |
| 5.5 质量分级 | T1/T2/T3 统计 + 风险提示 |
| 6 Prompt 构建 | 查询词 + 论文数 + Prompt 长度 |
| 7 报告撰写 | 报告草稿路径 + 概要 |
| 8 保存报告 | 最终报告路径 |

> **用户可以在 Stage 1 确认后说"继续执行完"**，此时 Claude Code
> 应一次性自动执行 Stage 2-8，中间不再暂停。

---

## WebSearch 失败处理

连续失败 ≥3 次：
1. 向用户报告："⚠️ WebSearch 服务不可用，无法自动获取领域趋势。"
2. 请求用户手动提供：该领域的 2-3 个最新研究方向、是否有特定子方向
3. 如果用户也无法提供，脚本可使用内置模板 fallback（但质量下降）
   - `stage1_demand.py` 会输出 ⚠️ 质量警告
   - Claude Code 应在后续报告中注明数据来源质量
