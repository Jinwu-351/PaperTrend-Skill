# 缺口分析指南（Gap Analysis Guide）

Stage 1 需求解析中，在构造关键词之前执行。目标：确保检索关键词覆盖该领域所有重要方向。

## 执行时机

在 Step 1（网络搜索）和 Step 2（用户澄清）**之后**、Step 3（构造关键词）**之前**。

---

## 三步缺口分析流程

### Step A: 识别主流方向（五轴分类法）

对任意研究领域，从以下 **五个轴** 系统性地识别主流方向，每个轴列出 1-3 个方向：

| 轴 | 含义 | 自问 |
|----|------|------|
| **理论轴** | 该领域的核心理论框架和数学模型 | "这个方向的研究生第一门课教什么理论？" |
| **方法轴** | 主流的实现手段和技术路线 | "综述论文里按什么方法论分类？" |
| **传感/数据轴** | 该领域依赖的传感器、数据源或输入模态 | "研究者用什么设备/数据做实验？" |
| **应用轴** | 该技术主要的落地场景和邻近社区 | "哪个应用领域产出了最多相关论文？" |
| **前沿轴** | 近 12-18 个月出现的新兴范式 | "最近顶会上出现的新热词是什么？" |

**方法**：基于 Step 1 网络搜索结果，对每个轴搜索验证 → 记录 2-3 个术语 → 汇总为 10-15 个候选方向 → 标注 P0/P1/P2 优先级。

**优先级定义**（领域无关）：

| 级别 | 定义 | 判定标准 |
|------|------|---------|
| **P0** | 必修方向 | 该领域 80% 以上综述论文会提及；或用户明确指定的子方向 |
| **P1** | 重要方向 | 独立的研究子社区，但规模小于 P0；或高度相关的邻近领域 |
| **P2** | 前沿/边缘方向 | 近 12 个月出现的新术语，尚未形成稳定社区；或跨领域迁移潜力大但验证少 |

### Step B: 关键词覆盖检查

将 Step A 识别的所有 P0/P1 方向与候选关键词逐一对照：

```
候选关键词列表:
  [1] "xxx"
  [2] "yyy"
  ...

对照矩阵 (方向 × 关键词):
                  kw1   kw2   kw3   kw4   kw5
  P0-方向A        ✅     -     -     -     -
  P0-方向B         -     -     ✅    -     -
  P1-方向C         -     -     -     -     ❌  ← 遗漏
  P1-方向D         -     ✅    -     -     -

覆盖率: P0 = 2/2 ✅  |  P1 = 1/2 ❌
```

### Step C: 关键词修正

对遗漏方向按优先级补充或替换：

1. **P0 遗漏** → **必须新增**关键词覆盖，不可跳过
2. **P1 遗漏** → 如关键词数量允许（<8 个），新增；否则替换最低价值的现有关键词
3. **P2 遗漏** → 仅当关键词 <8 且该 P2 方向与用户需求高度相关时才新增

**关键词价值评估**（替换决策用）：价值 = 领域区分度 × 搜索可得性。优先替换价值最低的关键词。

---

## 关键词质量规则（领域无关）

### 有效性检查（每个关键词必须满足）

| 规则 | 判定方法 |
|------|---------|
| **学术术语性** | 是否为该领域的标准英文术语？非日常用语，非自造缩写 |
| **搜索可得性** | 在 Semantic Scholar `/paper/search?query=...` 返回 ≥5 篇 |
| **领域区分度** | 搜索结果是否集中于目标领域而非泛化到无关领域？ |

### 语义重叠检查

| 判定维度 | 阈值 | 动作 |
|---------|------|------|
| 核心名词短语重叠率 | ≥50% | 合并为更精确的关键词 |
| 前 10 篇搜索结果重合率 | ≥40% | 保留区分度更高的那个 |

### 关键词数量规则

| 数量 | 条件 |
|------|------|
| **5 个** | 最低要求，至少覆盖所有 P0 方向 |
| **6-7 个** | 标准配置，覆盖 P0 + 主要 P1 |
| **8 个** | 上限，仅当每个关键词通过有效性+重叠检查 |

## 输出格式

在构造关键词时，将以下信息写入 `web_summary` 末尾：

```
## 缺口分析

五轴扫描:
  理论轴: [方向1, 方向2]
  方法轴: [方向3, 方向4]
  传感轴: [方向5]
  应用轴: [方向6]
  前沿轴: [方向7]

覆盖结果:
  P0 (已覆盖 3/3): [方向A, 方向B, 方向C]
  P1 (已覆盖 2/3): [方向D, 方向E]
  P1 (未覆盖 1/3): [方向F] — 因关键词已饱和(8个)，该方向在arXiv覆盖较少
  P2 (已覆盖 1/2): [方向G]

语义合并: "kw1" + "kw2" → "kw1_new" (核心词重叠 60%)
```

## 示例：绳驱机器人感知的缺口分析

以下仅为示意，实际执行时应基于网络搜索结果动态生成。

**五轴扫描结果**（基于网络搜索 + 领域知识）：

| 轴 | 识别的方向 | 优先级 |
|----|-----------|--------|
| 理论轴 | Cosserat Rod 建模、常曲率假设 | P0 |
| 方法轴 | 视觉伺服、物理储备池计算 | P0 |
| 传感轴 | FBG 形状传感、驱动侧力感知 | P0 |
| 应用轴 | 医学内窥镜、工业管道检测 | P0 |
| 前沿轴 | 神经隐式表示、事件相机 | P2 |

**覆盖检查** → 发现遗漏：FBG sensing、medical endoscopy、neural fields → 新增 3 个关键词。

**语义重叠检测** → "cable-driven robot perception" 与 "tendon-driven visual sensing" 重叠 60% → 合并保留前者。

**最终关键词（8 个）**：覆盖 P0 全部 + P1 部分 + P2 部分。
